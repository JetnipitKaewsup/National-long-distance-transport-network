# Architecture Specification
## Undergraduate Computer Networks Project

### 1. Project Objective
To design and study a **reliable and low-latency communication network** for long-distance transportation
using **modern computer networking concepts**.

This project focuses on:
- Network architecture design
- Reliability and redundancy
- Mobility and handover concepts
- Emergency communication scenarios

### 2. System Overview
The system is designed as a **Three-Layer Network Architecture** to demonstrate
how different networks can work together.

### 3. Network Architecture

#### Layer 1: Terrestrial Network (Primary)
- Technology: 5G (conceptual), Ethernet, IP
- Role:
  - Main communication channel
  - Low latency and stable connection
- Concepts studied:
  - Packet switching
  - Handover
  - Quality of Service (QoS)

#### Layer 2: Satellite Network (Backup)
- Technology: Satellite Internet (conceptual)
- Role:
  - Backup when terrestrial network is unavailable
  - Wide-area coverage
- Concepts studied:
  - Latency comparison
  - Network redundancy

#### Layer 3: Emergency Drone Network
- Technology: Wireless Mesh Network
- Role:
  - Temporary network in disaster areas
- Concepts studied:
  - Ad-hoc networks
  - Mesh routing
  - Coverage extension

### 4. Key Networking Concepts
- IP addressing
- Network handover
- Latency and jitter
- Redundancy and fault tolerance
- Basic network security concepts

### 5. Learning Outcomes
Students will be able to:
- Design layered network architectures
- Explain trade-offs between latency and coverage
- Understand emergency and backup networking
